SatDump builds from source on all major Linux distributions. Prebuilt x64 binaries for Ubuntu are available on the [Releases page](https://github.com/altillimity/SatDump/releases) if you prefer to skip compilation.

## Build NNG from source

If NNG is not available as a package for your distribution (Alpine, Gentoo, or older Debian/Ubuntu), build it from source before proceeding:

```bash
git clone https://github.com/nanomsg/nng.git -b v1.9.0
cd nng
mkdir build && cd build
cmake -DCMAKE_BUILD_TYPE=Release -DBUILD_SHARED_LIBS=ON -DCMAKE_INSTALL_PREFIX=/usr ..
make -j4
sudo make install
cd ../..
rm -rf nng
```

## Build SatDump

Once dependencies are installed, clone the repository and build:

```bash
git clone https://github.com/SatDump/SatDump.git
cd SatDump
mkdir build && cd build
cmake -DCMAKE_BUILD_TYPE=Release -DCMAKE_INSTALL_PREFIX=/usr ..
make -j`nproc`
```

<Note>
  On Raspberry Pi 3 (any revision) and older, use `make -j1` instead of `make -j\`nproc\`` — using multiple jobs will crash the system.
</Note>

### Run without installing

To run SatDump directly from the build directory, create symlinks so it can locate its resources and configuration:

```bash
ln -s ../resources .        # Symlink resources so it can run
ln -s ../satdump_cfg.json . # Symlink settings so it can run

./satdump-ui
```

### Install system-wide

```bash
sudo make install
```

## CMake build flags

You can customize the build by passing additional flags to the `cmake` command.

| Flag | Effect |
|---|---|
| `-DBUILD_GUI=OFF` | Build the CLI version only, omitting the GUI |
| `-DPLUGIN_HACKRF_SDR_SUPPORT=OFF` | Disable a specific SDR plugin (replace `HACKRF` with the plugin name) |
| `-DCMAKE_INSTALL_PREFIX=/usr` | Set the installation prefix |
