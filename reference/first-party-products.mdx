SatDump can process pre-existing official data files obtained directly from space and meteorological agencies, without requiring a live SDR capture. These are called first-party products — they are the authoritative Level 1 data files distributed by the agencies themselves, and SatDump handles the decoding and visualization for you.

<Note>
  This module is experimental and may contain bugs. You are encouraged to test it and report any issues on the [SatDump GitHub issue tracker](https://github.com/SatDump/SatDump/issues).
</Note>

## How to use first-party products

All first-party products must be processed through a single pipeline:

> **Data Stores/Archives Formats (NOAA/EUMETSAT/CMA/NASA), Experimental**

Point this pipeline at the directory or file you downloaded from the agency's data portal, and SatDump will detect the product format and decode it automatically.
