SatDump runs on Linux, Windows, macOS, Android, and inside Docker containers. The fastest path to a working installation is to download a pre-built release. If you want the latest code or need to customize which SDR drivers are compiled in, build from source.

<Note>
  Pre-built releases for Windows and macOS are available on the [GitHub Releases page](https://github.com/SatDump/SatDump/releases). Linux users are recommended to build from source for the best driver compatibility.
</Note>

<Tabs>
  <Tab title="Linux">

Linux builds from source are recommended for full SDR device support. Pre-built x64 Ubuntu binaries are available on the [releases page](https://github.com/SatDump/SatDump/releases) if you prefer not to compile.

<Steps>
  <Step title="Install dependencies">

Choose the command set for your distribution.

**Debian / Ubuntu / Debian-based:**

```bash
sudo apt install git build-essential cmake g++ pkgconf libfftw3-dev libpng-dev \
                 libtiff-dev libjemalloc-dev libcurl4-openssl-dev libsqlite3-dev
sudo apt install libvolk-dev          # use libvolk2-dev or libvolk1-dev if not found
sudo apt install libnng-dev           # see note below if not found

# Optional — include these for the features you need:
sudo apt install libglfw3-dev libdbus-1-dev   # GUI version
sudo apt install portaudio19-dev              # audio output
sudo apt install libzstd-dev                  # ZIQ recording compression
sudo apt install libhdf5-dev                  # EUMETSAT official products
sudo apt install librtlsdr-dev libhackrf-dev libairspy-dev libairspyhf-dev  # live SDR processing
sudo apt install libad9361-dev libiio-dev     # PlutoSDR, PlutoPlus, AntSDR
sudo apt install libbladerf-dev               # BladeRF
sudo apt install ocl-icd-opencl-dev           # OpenCL (recommended, speeds up some operations)
sudo apt install libarmadillo-dev
```

One-liner to install everything at once:

```bash
sudo apt install git build-essential cmake g++ pkgconf libfftw3-dev libpng-dev \
                 libtiff-dev libjemalloc-dev libcurl4-openssl-dev libvolk-dev libnng-dev \
                 libglfw3-dev zenity portaudio19-dev libzstd-dev libhdf5-dev librtlsdr-dev \
                 libhackrf-dev libairspy-dev libairspyhf-dev libad9361-dev libiio-dev \
                 libbladerf-dev libomp-dev ocl-icd-opencl-dev intel-opencl-icd mesa-opencl-icd \
                 libdbus-1-dev libarmadillo-dev libsqlite3-dev
```

**Red Hat / Fedora / RHEL-based:**

```bash
sudo dnf install git cmake g++ fftw-devel volk-devel libpng-devel jemalloc-devel \
                 libtiff-devel nng-devel curl-devel sqlite-devel

# Optional:
sudo dnf install glfw-devel zenity dbus-devel   # GUI version
sudo dnf install portaudio-devel                # audio output
sudo dnf install libzstd-devel                  # ZIQ compression
sudo dnf install hdf5-devel                     # EUMETSAT products
sudo dnf install rtl-sdr-devel hackrf-devel airspyone_host-devel  # live SDR
sudo dnf install ocl-icd                        # OpenCL
```

**openSUSE:**

```bash
sudo zypper install git gcc-c++ cmake g++ pkgconf fftw3-devel libpng-devel \
                    libtiff-devel jemalloc-devel libcurl-devel volk-devel  \
                    nng-devel sqlite3-devel

# Optional:
sudo zypper install libglfw-devel zenity dbus-1-devel   # GUI version
sudo zypper install portaudio-devel                     # audio output
sudo zypper install libzstd-devel                       # ZIQ compression
sudo zypper install hdf5-devel                          # EUMETSAT products
sudo zypper install rtl-sdr-devel hackrf-devel airspy-devel airspyhf-devel  # live SDR
sudo zypper install libad9361-iio-devel libiio-devel    # PlutoSDR family
sudo zypper install bladeRF-devel                       # BladeRF
sudo zypper install opencl-headers libOpenCL1           # OpenCL
```

**Alpine:**

```bash
# The testing repository is required for libvolk-dev. Build libnng from source (see note below).
sudo apk add git cmake make g++ pkgconf fftw-dev libvolk-dev libpng-dev \
             jemalloc-dev tiff-dev curl-dev sqlite-dev

# Optional:
sudo apk add glfw-dev zenity dbus-dev             # GUI version
sudo apk add portaudio-dev                        # audio output
sudo apk add zstd-dev                             # ZIQ compression
sudo apk add hdf5-dev                             # EUMETSAT products
sudo apk add librtlsdr-dev hackrf-dev airspyone-host-dev airspyhf-dev  # live SDR
sudo apk add opencl-dev                           # OpenCL
```

**Gentoo:**

```bash
sudo emerge --ask dev-vcs/git sci-libs/fftw dev-libs/jemalloc sci-libs/volk \
                  media-libs/libpng media-libs/tiff net-misc/curl dev-db/sqlite

# Optional:
sudo emerge --ask media-libs/glfw gnome-extra/zenity sys-apps/dbus   # GUI
sudo emerge --ask media-libs/portaudio                               # audio
sudo emerge --ask app-arch/zstd                                      # ZIQ compression
sudo emerge --ask sci-libs/hdf5                                      # EUMETSAT products
sudo emerge --ask net-wireless/rtl-sdr net-libs/libhackrf            # live SDR
sudo emerge --ask net-wireless/bladerf                               # BladeRF
sudo emerge --ask net-libs/libiio net-libs/libad9361-iio             # PlutoSDR
sudo emerge --ask dev-libs/opencl-icd-loader                         # OpenCL
```

  </Step>

  <Step title="Build NNG from source (if needed)">

If `libnng-dev` is not available in your distro's package manager, build it from source:

```bash
git clone https://github.com/nanomsg/nng.git -b v1.9.0
cd nng
mkdir build && cd build
cmake -DCMAKE_BUILD_TYPE=Release -DBUILD_SHARED_LIBS=ON -DCMAKE_INSTALL_PREFIX=/usr ..
make -j4
sudo make install
cd ../..
rm -rf nng
```

  </Step>

  <Step title="Clone and build SatDump">

```bash
git clone https://github.com/SatDump/SatDump.git
cd SatDump
mkdir build && cd build

# To skip the GUI, add -DBUILD_GUI=OFF
# To disable a specific SDR, add -DPLUGIN_HACKRF_SDR_SUPPORT=OFF or similar
cmake -DCMAKE_BUILD_TYPE=Release -DCMAKE_INSTALL_PREFIX=/usr ..
make -j`nproc`
```

<Warning>
  On Raspberry Pi 3 or older, use `make -j1` instead of `make -j\`nproc\`` to avoid crashing the system.
</Warning>

  </Step>

  <Step title="Run or install">

To run SatDump from the build directory without installing:

```bash
# Symlink required resources
ln -s ../resources .
ln -s ../satdump_cfg.json .

# Launch the GUI
./satdump-ui
```

To install system-wide:

```bash
sudo make install
```

  </Step>
</Steps>

  <Tab title="Windows">

<Steps>
  <Step title="Download the installer or portable app">

Go to the [GitHub Releases page](https://github.com/SatDump/SatDump/releases) and download either:

- **SatDump installer** — installs SatDump system-wide with a start menu entry
- **Portable app** — a zip you can extract and run anywhere without installing

Builds are compiled with Visual Studio 2019 for x64.

  </Step>

  <Step title="Install the Visual C++ Runtime (if needed)">

SatDump requires the Visual C++ Redistributable. It is likely already installed on your system, but if SatDump fails to launch, download it from [Microsoft's support page](https://support.microsoft.com/en-us/topic/the-latest-supported-visual-c-downloads-2647da03-1eea-4433-9aff-95f26a218cc0).

  </Step>

  <Step title="Launch SatDump">

Run **`satdump-ui.exe`** for the graphical interface or **`satdump.exe`** for the CLI.

  </Step>
</Steps>

<Note>
  Mingw builds are not supported — VOLK will not function correctly. For compilation instructions, see the [Windows platform page](/platforms/windows).
</Note>

  </Tab>

  <Tab title="macOS">

<Steps>
  <Step title="Download the pre-built app">

Dependency-free macOS builds are provided on the [GitHub Releases page](https://github.com/SatDump/SatDump/releases). Download the `.dmg` or `.zip`, open it, and drag SatDump to your Applications folder.

  </Step>

  <Step title="(Optional) Build from source with Homebrew">

If you want to build from source, [Homebrew](https://brew.sh) is required.

**Install dependencies:**

```bash
brew install cmake dylibbundler libtool autoconf automake meson orc libxml2 zlib \
             portaudio boost libusb nng volk libjpeg-turbo libtiff libomp libpng \
             glfw3 fftw jemalloc nng zstd armadillo hdf5 sqlite3
```

**Clone and build:**

```bash
git clone https://github.com/SatDump/SatDump.git && cd SatDump
mkdir build && cd build

# Build additional bundled dependencies
../macOS/build_deps.sh

# Build SatDump (add -DBUNDLING_MODE=ON if you want a distributable .app bundle)
cmake -DCMAKE_BUILD_TYPE=Release ..
make -j$(sysctl -n hw.logicalcpu)
```

**Run from the build directory:**

```bash
ln -s ../pipelines .
ln -s ../resources .
ln -s ../satdump_cfg.json .
./satdump-ui
```

**Or bundle into a .app:**

```bash
cmake -DCMAKE_BUILD_TYPE=Release -DBUNDLING_MODE=ON ..
make -j$(sysctl -n hw.logicalcpu)
../macOS/bundle.sh
# Bundled app is in ./MacApp
```

  </Step>
</Steps>

  <Tab title="Android">

<Steps>
  <Step title="Install from F-Droid (recommended)">

The preferred installation source for Android is [F-Droid](https://f-droid.org/packages/org.satdump.SatDump). F-Droid provides automatic updates and does not require a Google account.

  </Step>

  <Step title="Install from GitHub Releases (alternative)">

If F-Droid is not available to you, APKs are published on the [GitHub Releases page](https://github.com/SatDump/SatDump/releases).

  </Step>
</Steps>

Nearly all SatDump features work on Android. SDR device support is limited by USB host capability and available drivers. Supported devices on Android:

- RTL-SDR
- Airspy
- AirspyHF
- LimeSDR Mini
- HackRF

<Note>
  For Android-specific notes and USB OTG setup, see the [Android platform page](/platforms/android).
</Note>

  <Tab title="Docker">

Docker is a good option if you want to isolate SatDump's build environment and libraries from your host OS. The provided Dockerfile uses a multi-stage build: a build image with `-dev` packages and a smaller runtime image.

<Steps>
  <Step title="Set your host user IDs">

Files written by the container will be owned by the host user when UIDs match. Set them in `.env` before building:

```bash
printf "HOST_UID=$(id -u)\nHOST_GID=$(id -g)\n" > .env
```

  </Step>

  <Step title="Create the shared directory">

```bash
mkdir -p srv
```

  </Step>

  <Step title="Build the images">

```bash
docker compose build --build-arg CMAKE_BUILD_PARALLEL_LEVEL=8
```

  </Step>

  <Step title="Run a shell inside the container">

```bash
docker compose run --rm -it satdump
```

You can pass a command directly at the end of this line, or set `COMMAND: satdump_sdr_server` in `.env` to run a specific command automatically.

  </Step>

  <Step title="(Optional) Run the SDR server as a background service">

```bash
docker compose up -d
docker compose logs -f
```

  </Step>
</Steps>

**Using the GUI (`satdump-ui`) under Docker:**

To forward the display via X11, uncomment the X11 socket mount in `docker-compose.yml`:

```yaml
- type: 'bind'
  source: '/tmp/.X11-unix'
  target: '/tmp/.X11-unix'
```

Then allow the Docker user to connect to your X server and launch the UI:

```bash
xhost +local:docker
docker compose run --rm -it satdump satdump-ui
```

On WSL2, use `/run/desktop/mnt/host/wslg/.X11-unix` as the source instead of `/tmp/.X11-unix`.

<Note>
  For Docker-specific configuration, see the [Docker platform page](/platforms/docker).
</Note>
